## 2. 项目背景与目标
### 2.1 背景
在签证、留学、境外开户等场景中，用户通常需要提交英文版银行流水证明。现有人工翻译流程成本高、耗时长，且格式一致性难以保障。

### 2.2 建设目标
本项目旨在实现中文银行流水 PDF 的自动化处理，输出结构化英文 Excel 文件，提升交付效率与准确性。

### 2.3 目标用户
- 办理英签、美签等需要英文流水材料的个人用户
- 提供材料代办服务的中小机构

### 2.4 范围边界
- 输入：中文银行流水 PDF（扫描件或电子版）
- 输出：英文 `.xlsx` 文件（可保留原字段顺序与主要格式）
- 不包含：银行电子回单验真、法律认证翻译盖章

## 3. 总体技术方案
### 3.1 技术选型
| 层级 | 默认方案 | 可选方案 |
|---|---|---|
| 编程语言 | Python 3.10+ | - |
| Web 界面 | Streamlit | Gradio |
| PDF 解析 | PyMuPDF（fitz） | pdfplumber |
| OCR 识别 | PaddleOCR | RapidOCR |
| 翻译引擎 | 智谱 `glm-4-flash`（免费） | DeepL API |
| 数据处理 | pandas | polars |
| Excel 导出 | openpyxl | xlsxwriter |
| 配置管理 | python-dotenv | pydantic-settings |
| 部署方式 | 阿里云函数计算/容器化部署 | Vercel（轻量演示） |

### 3.2 架构概览
```
用户上传 PDF
    -> 文档预处理（页拆分/纠偏/去噪）
    -> 文本抽取（优先文本层，失败则 OCR）
    -> 字段结构化（日期/金额/余额/对方户名）
    -> 翻译与术语标准化
    -> 结果校验（金额格式、日期格式、行完整性）
    -> Excel 生成与下载
```

### 3.3 模块划分
1. `pdf_parser.py`：PDF 页处理、文本层提取、图片渲染。
2. `ocr_recognizer.py`：OCR 识别与版面块合并。
3. `statement_structurer.py`：流水字段解析与标准化。
4. `translator.py`：术语翻译、上下文翻译、缓存复用。
5. `excel_generator.py`：多 Sheet 输出、样式与格式控制。
6. `app.py`：上传交互、任务状态、结果下载。
7. `config.py`：API 密钥、阈值参数、环境配置。

## 4. 核心流程设计
### 4.1 处理流程
1. 用户上传 PDF，系统进行文件合法性校验（格式、大小、页数）。
2. 尝试直接提取文本层；若文本稀疏或缺失，自动切换 OCR。
3. 对识别结果执行结构化解析，映射为统一字段模型。
4. 按“固定术语优先 + 自由文本补充”的策略执行翻译。
5. 对关键字段进行格式与一致性校验。
6. 生成 Excel 并提供下载。

### 4.2 关键参数建议
| 参数 | 建议值 | 说明 |
|---|---|---|
| 文件大小上限 | 10 MB | 控制单任务资源占用 |
| 页数上限 | 30 页 | 防止超时与费用失控 |
| OCR 语言 | 简体中文 | 主流程默认 |
| 翻译目标语言 | 英文 | 后续可扩展多语言 |
| 接口超时 | 30 s | 翻译 API 请求超时 |
| 重试次数 | 2 次 | 网络波动兜底 |

## 8. 智谱 GLM-4-Flash API 接入说明

### 8.1 API 概览
- **模型名称**：`glm-4-flash`
- **API 地址**：`https://open.bigmodel.cn/api/paas/v4/chat/completions`
- **认证方式**：Bearer Token
- **费用**：免费（每个账号有初始配额）

### 8.2 Python 调用示例
```python
import requests

API_KEY = "your-api-key-here"
URL = "https://open.bigmodel.cn/api/paas/v4/chat/completions"
HEADERS = {
    "Authorization": f"Bearer {API_KEY}",
    "Content-Type": "application/json"
}

def translate(text: str, source_lang="中文", target_lang="英文") -> str:
    payload = {
        "model": "glm-4-flash",
        "messages": [{"role": "user", "content": f"将{source_lang}翻译成{target_lang}: {text}"}]
    }
    response = requests.post(URL, headers=HEADERS, json=payload, timeout=30)
    return response.json()["choices"][0]["message"]["content"]

# 示例
translate("交易日期")  # -> "Transaction Date"
```

### 8.3 费用估算
- 单页流水（~50条交易）：免费
- 10页流水：免费
- 100次/天：免费

> ⚠️ 免费配额用完后需付费

### 8.4 稳定性建议
- 重试机制：网络波动时自动重试2-3次
- 限流处理：429响应时指数退避
- 熔断降级：连续失败时切换本地术语字典

## 11. 风险与应对
| 风险 | 影响 | 应对措施 |
|---|---|---|
| 银行版式差异大 | 字段识别不稳定 | 引入模板规则层 + 置信度回退 |
| OCR 误识别 | 翻译与导出错误 | 关键字段二次校验（日期/金额正则） |
| 翻译 API 限流 | 任务失败或延迟 | 本地术语字典 + 请求重试 + 队列削峰 |
| 隐私数据泄露风险 | 合规风险高 | 脱敏日志 + 临时文件清理 + 权限隔离 |
