## 代理问题

### 问题描述
- 使用 OpenAI Codex 模型需要代理
- 但全局代理会导致飞书断联

### 尝试过的方案
1. OpenClaw 配置 env 全局代理 → ❌ 飞书断联
2. env + NO_PROXY → ❌ 还是断联
3. Codex 未登录 → ✅ 今天发现，已修复
4. HTTP 代理 (10900) → ❌ 401 认证错误
5. SOCKS5 代理 (10011) → ❌ 连接失败

### Codex 建议的新方案
**方案 1：终端级别代理 + NO_PROXY**
```powershell
$env:HTTP_PROXY="http://127.0.0.1:10090"
$env:HTTPS_PROXY="http://127.0.0.1:10090"
$env:ALL_PROXY="socks5://127.0.0.1:10011"
$env:NO_PROXY="localhost,127.0.0.1,.feishu.cn,.larksuite.com,open.feishu.cn"
```

**方案 2：QuickQ 规则模式（推荐）**
- 在 QuickQ 中开启"规则模式"或"分流模式"
- OpenAI 自动走代理，飞书自动直连
